# 参考図台帳検索システム — Access → MembrySubSystem 移行 設計書（v0 ドラフト）

営業支援ツール「参考図台帳検索システム」(Microsoft Access `参考図.accdb`) を MembrySubSystem
(Next.js App Router / Lark Bitable / AWS Amplify) へ移行する。本書は現行Accessの調査結果と
移行設計（案）。**Box(PDF)アクセス方式など未確定の論点は §8 に集約**。確定後に実装着手する。

> 調査日: 2026-06-29 / 元ファイル: `C:\Users\tatsuma.m\OneDrive\ドキュメント\参考図.accdb`(約21MB)
> 取得方法: `Microsoft.ACE.OLEDB.16.0` 経由でスキーマ・件数・サンプルを取得。

---

## 1. 目的・背景

- 過去の参考図面（PDF）を、建屋仕様（用途・寸法・部材・基準風雪 等）で検索し、該当図面を開く
  営業支援ツール。現状は Access アプリ＋Box内PDF。
- Access は配布・同時利用・保守に難。MembrySubSystem の営業メニューへ統合し、
  「ブラウザで条件を選ぶ → 一覧 → PDFを開く」をWeb化する。
- 既存の `基準風速・垂直積雪量検索`(§営業)と同じ実装規約・登録フローに揃える。

### 2.3 PDF格納（重要）

- **PDFは全件、単一のBoxフォルダに格納**。ファイルパスの種類は1つだけ:

## 5. API設計

- `GET /api/eigyo/sankou-zu`：Larkから台帳＋部品マスタを全件取得→整形→**モジュールキャッシュ**
  （4,268行≒9ページ。kijun-fusoku と同方式。県単位のような分割は不要だが件数次第で初回数秒許容）。
- `GET /api/eigyo/sankou-zu/file?...`：**PDFをサーバ経由で解決/中継**（§8-Aで Box API採用時）。
  Boxトークンはサーバ側のみで保持しクライアントに出さない（既存 `gsi-elevation` と同方針）。

## 8. 未確定の論点（要判断）

### A. PDF(Box)アクセス方式 ★決定 = Box API（サーバ経由・実行時）

サーバが Box Platform アプリでフォルダ内をファイル名検索→file_id→プレビュー/ダウンロードURLを発行。
Boxトークンはサーバ側のみ・クライアントに出さない。権限制御可。実装規約は既存 `gsi-elevation` と同方針。

**実装方針**:
- `GET /api/eigyo/sankou-zu/file?name={ファイル名}`（要セッション）。
  サーバが Box でフォルダ(固定 folder_id)内を `name` で検索→ヒットした file_id の
  ダウンロード/プレビューURLへ302、または `/embed` プレビューを返す。ファイル名重複時は最新/先頭を採用。
- 認証は Box の **サーバ認証**を想定: JWT または Client Credentials Grant(CCG)。
  Box App の認証情報・対象 folder_id はサーバ環境変数で管理（`.env`/Amplify、クライアント非公開）。

**移行前に用意が必要（依頼事項）**:
1. **Box Platform アプリ**（推奨: CCG もしくは JWT のサーバ認証アプリ）。
   発行物 = Client ID / Client Secret（JWTなら公開鍵ペア＋設定JSON）、Enterprise ID。
2. **アプリの認可**: Box管理コンソールでアプリを承認（admin操作）。
3. **対象フォルダへのアクセス権**: サービスアカウントを
   `共有フォルダ/system/SubSystem/参考図面データ` の**コラボレーター(閲覧者)に追加** or 当該folderの folder_id を共有。
4. その **folder_id**（BoxのURL末尾の数値）。

→ これらが揃えばファイル中継APIを実装可能。揃うまではBox非依存の作業（データ移行・検索UI/API）を先行できる。

### B. データ保持先 base

- master base（全環境共有） or project base のどちらに置くか（kijun-fusoku は project base に手動作成）。

### C. 受注情報・売約引当の扱い ★決定 = 移行しない

- 案件一覧は既にMembrySubSystem側に存在するため受注情報は移行しない（§3.3）。売約引当はv2で検討。

### D. スコープ/権限

- ルート名（`/eigyo/sankou-zu` で良いか）・メニュー表示名・営業メニュー内の表示順・利用権限範囲。
- 検索項目の優先度（v1必須＝用途/建屋区分/寸法/フリーワード、部材検索はv2 等の段階分けの是非）。
