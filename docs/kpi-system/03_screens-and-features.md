### UI

## ③ 施策管理(PDCA) `/seisan-kpi/measures`

Excel `03_施策管理` 相当。部署グループ単位の重点施策と月次PDCA。

> **データ構造の要点(確定)**: **重点施策 : PDCA = 1 : 多**。1つの施策(`SEISAN_KPI_MEASURE`)に、月ごとのPDCAログ(`SEISAN_KPI_PDCA`)が複数ぶら下がる。**施策は件数無制限**(2件固定ではない)・期途中の追加変更可。

> **グループ単位の表示(確定)**: 施策・主要KPIは **グループ**(`SEISAN_KPI_GROUP`)単位で表示する。グループは部署を束ねる集約レンズで、**1部署が複数グループに重複所属できる**(M:N / `SEISAN_KPI_GROUP_MEMBER`)。
> - 例: **鉄工課グループ** = 本社鉄工課 + 第二鉄工課 + 北関東鉄工課 ／ **北関東工場グループ** = 北関東鉄工課 + 北関東縫製課。
> - **北関東鉄工課は両グループに現れる**(重複表示)。実績は北関東鉄工課として1回だけ登録され、グループ表示時に参照(複製なし)。
> - グループ定義(`⑤ KPIマスタ管理`内の「グループマスタ管理」)で、管理者が自由にグループ作成・部署の所属変更が可能(組織変更に追従)。

### UI — マスター・ディテール型

施策が可変件数 × 各施策に1:NのPDCAがぶら下がるため、**施策一覧(マスター)＋ 選択施策のPDCA履歴(ディテール)** の2ペイン構成とする(モック `mockups/seisan-kpi-measures.html` 参照)。

```
[部署グループ ▼: 製造部 鉄工課]   [期 ▼: 50期]

▼ この部の主要KPI(現在の達成状況)   ※自動表示
  クレーム: 4/目標6 🟢   社内不具合: 24/目標35 🟢  …

┌── マスター: 重点施策一覧(全N件) ──┐  ┌── ディテール: 選択施策のPDCA履歴 ──┐
│ No 施策名/対象KPI  状態  基準→現在 判定│  │ 施策1: 鉄工チェックリスト           │
│ 1  鉄工ﾁｪｯｸﾘｽﾄ   実施中 13→9  🔴 ◀選│  │ 対象KPI: 本社鉄工課 社内不具合       │
│ 2  3週間工程運用  実施中 19.5→21 🟡   │  │ 月  計画 実施 KPI実績 効果 ｺﾒﾝﾄ ｱｸｼｮﾝ│
│ 3  外注検査二重化 完了  166→144 🔴   │  │ 8月 …   …   11(-15%) 改善 …  継続 │
│ … (件数無制限)                       │  │ 9月 …   …   10(-9%)  改善 …  強化 │
│ [＋施策追加]                         │  │ 10月 (会議前に記入)  自動:横ばい …  │
└──────────────────────────────────┘  └──────────────────────────────────┘
   ▲ 行をクリックで選択 → 右のPDCA履歴が切替
```

- **マスター(左)**: 施策を行で一覧。状態(実施中/完了/中止/下書き)・基準値→現在値・狙い値・判定・PDCA件数。`＋施策追加`で無制限に追加。
- **ディテール(右)**: 選択中の**1施策のPDCA全月**(1:N)を時系列表示。施策を切り替えると履歴も切替。
- これにより「施策が何件でも」「各施策のPDCA履歴を辿りながら」運用できる。

### ポイント

- **① 施策の追加・変更・終了**: `＋施策追加` で期途中でも追加可。各施策に **状態(下書き/実施中/完了/中止)** と **開始月/終了月** を持ち、ライフサイクルを管理。施策名・対象KPIの変更、完了/中止への状態変更が可能。全操作は AUDIT 記録。
- **③ 対象KPI(目標値)の明示**: 対象KPIドロップダウンは MASTER から該当グループの主要KPIを候補表示。選択で ACTUAL から **実績/目標/判定を自動表示**(Excel「KPI参照マスタ」をAPI化)。さらに施策固有の **狙い値**(例: 不具合 9→5)と **基準値**(開始時実績)を併記し、「何の数字をどこまで改善する施策か」を一目化。
- **④ 効果判定基準(明確化)**: 効果は対象KPI実績の変化で**自動判定(改善/横ばい/悪化)** を提示 → 責任者が確定(手動上書き可)。
  - 改善=良い方向に5%以上改善 or 判定ランク上昇、横ばい=±5%以内、悪化=逆方向5%以上 or ランク低下(`02_data-model.md` §2.6)。
  - PDCA行に**対象KPIの実績推移ミニグラフ**を併記し、判定根拠を可視化(主観排除)。
- **翌月アクション**: 継続/強化/見直し/完了 の選択。
- **役割別の編集領域**:
  - 計画/実施/効果 → 各部責任者(部長・次長・課長)
  - 本部長コメント/翌月アクション → 本部長(会議で記入)
- 色分け(会議前=青系、会議中=橙緑)を凡例表示。

### ② 過去施策の参照(施策履歴)

- **期セレクタ**で過去期(49期以前も移行分があれば)の施策・PDCAを遡って閲覧。
- **施策一覧/履歴タブ**: 部署グループ・対象KPI・状態(完了/中止含む)・期 で横断検索。
  - 「このKPIに対して過去どんな施策を打ち、効果はどうだったか」を時系列で確認。
  - 完了施策の最終効果(改善/横ばい/悪化)と翌月アクション履歴を一覧表示。
- 同一KPIに対する**施策の打ち手の蓄積**をナレッジとして活用(再発時の参考)。

## ④ ★達成評価 `/seisan-kpi/stars`

Excel `04_製造部★達成表` / `05_間接部門★達成表` 相当。モック: `mockups/seisan-kpi-stars.html`。

> **★達成評価は部署ごと(確定)**: 集計単位は **Lark部署**(製造部6課 + 間接3課)。グループ(`SEISAN_KPI_GROUP`)集約ではなく、部署単位で★を算出・表示する。

### UI

- **タブ**: 製造部(6課) / 間接部門(調達・管理・検査)。
- **★達成ランキング**: 部署ごとの総合計★を降順表示(🥇🥈🥉)。
- **表(行=部署×項目、列=8月〜7月+合計★)**:
  - 各セルに★/空白(月間目標達成で★)。`SEISAN_KPI_ACTUAL` と月間目標の比較で自動判定。
  - 部署ごとの「合計★数(自動)」行。
  - **5S大賞 / 労災**行は手入力(`SEISAN_KPI_STAR_ADJ`)。
  - 「総合計★数(5S・労災込)」= 自動★ + 期末ボーナス(年間達成で★+3) + 手入力調整。
- 間接部門の特例(経過月内の空欄も達成扱い)をロジックに反映。
- 部署は Lark Contact 部門を参照(改称・組織変更に追従)。
- ランキングはダッシュボードと連動。

## ⑤ KPIマスタ管理 `/seisan-kpi/master`

Excel `07_KPIマスタ` の編集。**管理者専用**(DX推進室)。

### UI

- 63指標の一覧(階層/部門/部署/カテゴリでフィルタ)。
- 編集項目: KPI名称・単位・集計タイプ・良い方向・年間目標・月次目標換算・KPIオーナー・データソース・入力タイミング・有効フラグ・備考。
- **期切替**: 新期作成(前期定義を複製 → 目標値だけ更新)機能。
- 組織変更対応: 部署の追加/無効化(有効フラグ)。
- 変更は AUDIT 記録。誤操作防止のため確認ダイアログ。

### グループマスタ管理(タブ)

- **グループの作成・編集**(`SEISAN_KPI_GROUP`): グループ名・種別(機能別/拠点別)・並び順・有効フラグ。
- **所属部署の設定**(`SEISAN_KPI_GROUP_MEMBER`): グループに部署を複数追加。**1部署を複数グループに所属**させられる(例: 北関東鉄工課 → 鉄工課グループ と 北関東工場グループ の両方)。
- チェックボックス式の所属マトリクス(行=部署 × 列=グループ)で重複所属を直感的に設定。
- 期ごとに構成変更可(組織変更に追従)。変更は AUDIT 記録。

## ⑥ 過去実績参照 `/seisan-kpi/history`

Excel `06_過去実績`。43-49期の推移 + 50期目標。モック: `mockups/seisan-kpi-history.html`。

### UI

- **抽出スコープ切替(タブ/セレクタ)**: **全社・部門 / 部署別 / グループ別** の3スコープで照会。
  - **全社・部門**: 集約指標(鉄工生産量/縫製生産量/品質/経営指標 等)の 43-49期推移 + 50期目標。
  - **部署別**: 部署(Lark部署)を選択 → その部署のKPI履歴(過去期実績 / 50期目標 / 50期現在 / 判定)。50期〜は `SEISAN_KPI_ACTUAL` から、43-49期は移行レガシーがあれば併記。
  - **グループ別**: グループを選択 → 所属部署(M:N)を**合算(累計系)/平均(率系)**した推移。北関東鉄工課が複数グループに属する場合は各グループで重複計上(表示上)。
- 各指標カテゴリごとの推移テーブル&折れ線グラフ。50期目標を併記し**目標の妥当性を視覚化**(過去レンジに対する位置=ストレッチ/妥当/要努力)。
- **データ源の明示**: 50期以降は当システムの部署別実績(時系列が充実)、43-49期は移行レガシー(集約中心)。部署別の深い時系列は運用を重ねるほど蓄積。
- 閲覧専用(編集は管理者のみ)。

## ❓要確認(画面分)

1. **グラフライブラリ**: 既存 package.json に recharts 等があるか要確認。なければ採用ライブラリを決定。
2. **モックアップ先行**: 要件 §7.2「最初に画面イメージで使い勝手をすり合わせ」。実装前に①ダッシュボード/②入力のモックを作成しレビューする工程を挟むか。
3. **会議投影モード**: フォント大・余白広の「プレゼンモード」を別途用意するか(ダッシュボードの全画面表示)。
4. **入力の同時編集**: 複数人同時入力時の競合制御(楽観ロック/最終更新者表示)の要否。
