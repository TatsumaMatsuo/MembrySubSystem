### 確定事項（2026-07-18）
- 出力＝**3ボタン**: PDF出力（DL）／EXCEL出力（DL）／**案件書庫保管（PDFのみ保存）**。
- 案件書庫の書類種別「**工事写真台帳**」を **工務課**カテゴリに新設（`DOCUMENT_CATEGORIES.工務課` へ追加）。工務課は既存カテゴリ。
- 追加依存 `jspdf`/`html2canvas`/`exceljs` を導入。
- 情報欄の工種＝日報「作業内容」を流用し画面編集（日報フォームは不変）。
- ⚠ 案件書庫の`書類種別`が単一選択の場合、選択肢「工事写真台帳」の追加(Lark UI)が必要な可能性→実装時に既存 `/api/documents/upload` の書込挙動で確認。

## 1. 目的・スコープ（指示ベース）

- 売約詳細画面に **「工事写真台帳」タブ**を追加する（既存の「作業日報」タブと同列）。
- 当該**製番（工事）の現場作業日報に登録された現場写真**を、**施工業者ごと**に取り込み、Web上で**工事写真台帳を作成**する。
- 顧客報告・社内検査・元請提出用の標準レイアウト台帳を、**画面で編集→プレビュー→PDF出力**まで行う。

### 要件定義書からの主な変更（指示優先）
| 要件定義書 | 本設計（Web版） |
|---|---|
| Excel VBA デスクトップツール | **Web（Next.js/売約詳細タブ）** |
| 抽出条件で任意テーブルを検索 | **現在の製番の現場作業日報に固定**。業者/期間で絞り込み |
| Excel生成＋PDF変換 | **Web上でA4レイアウト作成→ブラウザ印刷でPDF**（Excel出力は将来オプション） |
| VBAでLark認証(app_secret)・写真DL | 既存の**サーバAPI経由**（認証はセッション。写真は `/api/file/proxy`）|
| テンプレートExcelファイル管理 | **画面のレイアウト選択（L1/L2/L3/L6）** |

## 2. データソース（既存資産の再利用）

- **写真・情報**: 現場作業日報。`GET /api/nippou?seiban=<製番>` が `reports`（`会社名`/`報告者氏名`/`作業報告日`/`作業内容`/`特記事項`/`現場写真[]`）を返す。売約詳細の作業日報タブと同様に**受付コード→施工業者でグルーピング**（`nippouGroups` 相当）。
- **写真バイト**: `GET /api/file/proxy?file_token=<t>&table_id=<NIPPOU>`（同一オリジンでinline配信。既存の施工業者別DLで実績あり）。台帳の`<img>`はこのURLを指す。
- **表紙情報**: `GET /api/baiyaku-detail?seiban=` から 工事名=`受注件名` / 施工場所=`納入先住所` / 発注者=`得意先宛名` 等。工事番号=製番。施工者=自社（固定/ロゴ）。作成日=当日。

## 3. 情報欄（黒板情報）フィールドマッピング

写真1枚ごとの情報欄を、日報レコードから初期構成し**画面で編集可能**とする。

| 台帳 情報欄 | 初期値（現場作業日報） | 編集 |
|---|---|---|
| 施工業者 | 会社名 | 可 |
| 撮影日 | 作業報告日 | 可 |
| 工種／撮影箇所・内容 | 作業内容 | 可 |
| 撮影者 | 報告者氏名 | 可 |
| 備考／コメント | 特記事項・連絡事項 | 可 |

※ 要件定義書の「工種・区分（単一選択）」「撮影箇所」は現場作業日報に専用項目が無いため、初期は`作業内容`を流用しユーザーが上書き。将来、日報フォーム項目追加で自動化可（課題）。

## 4. 画面・操作（UX）

売約詳細「工事写真台帳」タブ内で完結。

1. **業者・対象選択**: 施工業者ごとにセクション表示。台帳化する業者を選択（複数可・既定は全業者）。任意で作業日From-To絞り込み。
2. **写真選択**: 各写真にチェック（既定=全選択）。並び順＝施工業者→撮影日昇順（既定）／登録順を選択可。
3. **情報欄編集**: 写真ごとの情報欄（§3）をインライン編集。空欄は非表示可。
4. **レイアウト選択**: L1/L2/L3(標準)/L6（A4縦・1/2/3/6枚/頁）。表紙ON/OFF、業者ごと改ページ（既定ON）。
5. **表紙編集**: 工事名・工事番号・工期・施工場所・発注者・施工者・作成日（初期値は§2、編集可）。
6. **プレビュー**: A4ページ区切りの**実寸プレビュー**（ヘッダ=工事名/工事番号、フッタ=ページ番号）。
7. **出力**: 「**PDF出力（印刷）**」ボタンで印刷用ビュー→ブラウザのPDF保存。※§7参照。

## 5. レイアウトテンプレート（要件定義書 F-07 準拠）

| パターン | 枚/頁 | 用紙 | 用途 |
|---|---|---|---|
| L1 | 1 | A4縦 | 全景・重要写真 |
| L2 | 2 | A4縦 | 施工前後の対比 |
| **L3（標準）** | 3 | A4縦 | 一般的な写真台帳（写真＋右/下に情報欄） |
| L6 | 6 | A4縦 | 枚数の多い工事 |

- 写真は枠に対し**縦横比保持で内接**（歪み・はみ出しなし）。縦写真は中央配置。
- 情報欄は写真ごとに§3項目を表示。文字超過は自動縮小（最小サイズあり）。

## 6. 表紙・グルーピング

- **表紙**: 工事名/工事番号/工期/施工場所/発注者/施工者(ロゴ)/作成日。
- **グルーピング**: 施工業者単位（既定）。業者ごとに見出し＋改ページ。目次は将来オプション。
- **並び順**: 業者内は撮影日昇順（既定）／登録順。

## 7. 出力方式（3ボタン・確定）

台帳プレビュー画面に**3つの出力ボタン**を用意する。

| ボタン | 動作 | 生成方式 |
|---|---|---|
| **PDF出力** | 台帳PDFをその場でダウンロード（WEBプレビューから） | クライアントで **jsPDF + html2canvas**（A4各ページのプレビューDOMを画像化してPDF化）→ Blob をダウンロード |
| **EXCEL出力** | 台帳Excel(.xlsx)をダウンロード | **exceljs**（写真を埋め込み配置。SheetJS `xlsx` は画像埋込非対応のため exceljs を採用） |
| **案件書庫保管** | 生成PDFを**案件書庫**へ保存 → 資料ダウンロードからDL可能に | 生成した同じPDF Blob を `POST /api/documents/upload`（`department`／`documentType="工事写真台帳"`）でアップロード |

- **PDF生成の基盤**: プレビューと同一DOMを `html2canvas` でページ単位に画像化し `jsPDF` でA4に貼る（見た目一致）。写真は `/api/file/proxy`（同一オリジン）なので html2canvas でCORS汚染なし。読込完了待ち。
- **案件書庫保管の保存先**: `PROJECT_DOCUMENTS`（tblHdcIy11FwCLCE, 行＝製番×部署×書類種別×添付）。**書類種別「工事写真台帳」を新設**（`DOCUMENT_CATEGORIES` にコード追加）。`資料ダウンロード`/`関連資料`から利用可能に（対象外設定はしない＝DL可）。
  - ⚠ `書類種別`が単一選択の場合、選択肢「工事写真台帳」を Lark UI で追加要（未登録だと 1254302）。テキスト型なら不要。実装前に確認。
- **ファイル名規則**: 既定 `<製番>_工事写真台帳_<作成日YYYYMMDD>.pdf/.xlsx`（要件定義書 F-11 準拠、可変）。
- 追加依存: `jspdf`, `html2canvas`, `exceljs`（package.json 追加）。`xlsx` は既存だが画像埋込不可のため Excel は exceljs。

## 8. 保存（ドラフト永続化）

- **v1**: 台帳ドラフト（選択・情報欄編集）の永続化は行わない。**出力の永続化は「案件書庫保管」**（生成PDFを案件書庫へ）で担保。開くたびに日報から再構成。
- **将来**: ドラフト（選択写真・情報欄上書き・レイアウト・表紙）の保存→再編集は必要なら別テーブルで対応（フェーズ2）。

## 10. 非機能・留意

- **Amplify(28s)**: 生成はクライアント主体（画像はブラウザ読込）でサーバ処理は薄い。大量写真時はページング済みの `/api/nippou` で取得。
- **HEIC**: ブラウザ表示不可のことがある→運用でJPEG徹底（要件定義書 課題1と同旨）。混在時はサムネイル不可を検知表示。
- **写真枚数**: 数百枚規模はDOM/印刷が重くなり得る→レイアウトとページ分割で緩和。極端な枚数は業者/期間で絞る運用。
- **権限**: 売約詳細内なので既存の閲覧権限に準拠（追加権限なし）。

## 11. 確定事項 / 残る確認点

**確定（2026-07-18）**:
- **出力＝3ボタン**: PDF出力（DL）／EXCEL出力（DL）／案件書庫保管（案件書庫へ保存し資料DL可）。
- 情報欄の工種＝作業内容を流用し編集可（日報フォームは変更しない）※未確定なら要指摘。

**残る確認点**:
1. **書類種別「工事写真台帳」**: 案件書庫の`書類種別`が単一選択なら選択肢追加（Lark UI）が必要。どの**部署カテゴリ**に置くか（例: 営業部 or 工務課）。
2. **案件書庫保管の対象**: PDFのみ保存か、Excelも保存するか。
3. **標準レイアウト**: 既定 L3（3枚/頁）。情報欄は写真の右/下どちらを既定に。
4. **表紙の工期**: 発注者＝得意先宛名でよいか。工期の取得元（施工開始日〜終了日？ 手入力可）。
5. **追加依存の可否**: `jspdf`/`html2canvas`/`exceljs` の追加でよいか。
