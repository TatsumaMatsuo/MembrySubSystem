# 参考図台帳検索 — Lark テーブル定義（作成依頼書）

MembrySubSystem のアプリは Lark Bitable の**テーブル/フィールドを新規作成できない**（既知制約・403
RolePermNotAllow）。以下の2テーブルを **Lark UI で手動作成**してください。作成後、テーブルID
（URLの `tbl…`）を共有いただければ、`lib/lark-tables.ts` に登録して投入スクリプトを実行します。

- **配置先 base**: project base（基準風速検索と同じ。全環境共有でよければ master でも可。要確定）
- **項目名は日本語**（業務ユーザーが直接見るため。Access の列名を踏襲）
- レコード件数: 参考図面台帳 4,268 / 参考図面部品マスタ 853

## テーブル1: `参考図面台帳`（参照図面情報2 由来 / 4,268行）

> Access の 61 列のうち **`ファイルパス` は作成不要**（全件が単一Boxフォルダのため。フォルダはアプリ側の
> 設定で保持し、行ごとは `ファイル名` だけで突合）。残り 60 列を作成。

### 数値型（10）
範囲検索に使うため数値。
`伝票番号`(主キー相当・必須) / `期` / `設計条件(基準風)` / `設計条件(基準雪)` / `間口` / `桁行` / `軒高` /
`柱ピッチ` / `勾配` / `庇出巾`

### テキスト型・複数行（9）
選択部品をまとめた複数行テキスト。
`計画概要memo` / `形状関連` / `出入口関連` / `膜関連` / `設備関連` / `構造関連` / `移動建屋関連` /
`開閉関連` / `畜舎関連`

### テキスト型・単一行（41）
`管理番号` / `管理名` / `売約番号` / `案件名` / `設計ルート` / `申請有無` / `建屋区分` / `用途` /
`出入口1` / `サイズ1` / `出入口2` / `サイズ2` / `壁面` /
`柱形状` / `B-PL形状` / `C1` / `柱成` / `柱ラチ` / `T1` / `梁成` / `梁ラチ` / `G1` /
`B1` / `B2` / `B3` / `B4` / `P1` / `P2` / `P3` / `P4` / `Ga` / `Gc` / `WB` / `ST` /
`基礎形状` / `F1` / `F2` / `F3` / `FG` / `土間` /
`ファイル名`  ← **PDF突合キー（Boxフォルダ内のファイル名）**

> 備考: `柱成`/`梁成` は元データで数値だが、部材コード的にも使うためテキストで作成（範囲検索しない）。
> 単一選択(ドロップダウン)は使わず**テキスト**で作成してください（アプリの列挙値とLark選択肢の完全一致が
> 必要になり保守が重くなるため。用途/建屋区分/設計ルート/申請有無 も当面テキスト）。

## PDF（Box）の扱い（参考・設計確定事項）

- PDFは全件 **単一Boxフォルダ** `…/共有フォルダ/system/SubSystem/参考図面データ/` に格納。
- 行→PDF は **多対一**（同一ファイルを複数行が参照）。台帳には `ファイル名` のみ保持。
- 実行時は `GET /api/eigyo/sankou-zu/file?name={ファイル名}` がサーバ経由でBox APIを呼び解決
  （Boxトークンはサーバのみ）。**Box Platform アプリ・folder_id の手配が別途必要**（README §8-A）。
