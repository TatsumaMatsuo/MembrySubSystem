# 02. データモデル設計(Lark Base)

現行 MembrySubSystem の規約(`lib/lark-tables.ts` の `getLarkTables()` / `TABLE_BASE_CONFIG` / `*_FIELDS` 定義)に合わせ、**project base** に新規テーブル群を追加する。

## 0. 設計方針

- **縦持ち(long format)を採用**: Excel は「KPI×12ヶ月列」の横持ちだが、Lark Bitable とフィルタ/集計の都合上、月次実績は **1レコード=1KPI×1ヶ月** の縦持ちにする。
- **保存値 + 読込時算出のハイブリッド**: `年累計/平均`・`進捗率`・`判定`・`★` は原則**保存せず読込時に算出**(Excel 数式の移植)。ただし監査・パフォーマンス目的でスナップショット保存も選択可(→ 実装計画で判断)。
- **期(fiscal period)でパーティション**: 全トランザクションテーブルに `期` を持たせ、期切替・複数期保持・過去参照を可能にする(拡張性要件)。
- **KPI_ID を業務キー**として全テーブルを連結(Excel の `M-xx` を踏襲)。
- **部署マスタ = Lark Contact 部門(確定)**: 独自テーブルを作らず、Lark の部署を正とする。既存 `lib/menu-permission.ts` の部門ツリー流用。改称・組織変更に自動追従。
- **実績登録は部門・部署単位 / ★達成評価は部署ごと(確定)**: グループ(`SEISAN_KPI_GROUP`)は施策・表示の集約レンズ。**★達成は部署単位**で算出(グループ集約ではない)。

## 2. テーブル定義詳細

### 2.0 経営レイヤー(L0/L1)

> **拡張性方針(確定)**: 中期経営計画は**複数本を追加・運用できる**構造にする。現中計(50→52期)に続けて、管理者がマスタにレコードを足すだけで 53→55期、56→58期… を**コード改修なしで追加**できる。そのため中計を **ヘッダ(計画単位)+ 明細(指標×期)** に正規化する。

#### 2.0.1a `KEIEI_MIDTERM_PLAN_HEADER`(中期経営計画ヘッダ)

中計1本を1レコードで表す。複数の中計を並存させ、画面の中計セレクタで切替。

| フィールド名 | 型 | 説明 | 例 |
|-------------|----|----|----|
| 中計ID | テキスト | 業務キー | `MTP-2` |
| 中計名 | テキスト | | `第◯次中期経営計画` |
| 開始期 | 数値 | 計画初年度 | `53` |
| 終了期 | 数値 | 計画最終年度 | `55` |
| ステータス | 単一選択 | 現行/次期/過去/ドラフト | `次期` |
| KGI指標セット | 複数選択 | この中計で追う指標 | `売上高,ROA,労働生産性` |
| 補間方法(既定) | 単一選択 | 線形補間/手動設定 | `線形補間` |
| 備考 | テキスト | | |

> 「現行」中計は同時に1本(ダッシュボード既定表示)。`開始期`〜`終了期` は任意スパン(3カ年に限定しない)。

#### 2.0.1b `KEIEI_MIDTERM_PLAN`(中期経営計画 明細 / 指標×期)

ヘッダにぶら下がる、指標×対象期の目標値。**線形補間で自動生成**(起点期→終了期のKGIを直線按分)し、各期 個別修正も可能。

| フィールド名 | 型 | 説明 | 例 |
|-------------|----|----|----|
| 明細ID | テキスト | 業務キー | `MTP-2-ROA-55` |
| 中計ID | テキスト | → HEADER 連結 | `MTP-2` |
| 指標 | 単一選択 | 売上高/ROA/労働生産性/粗利率/… | `ROA` |
| 単位 | テキスト | | `億円` / `%` / `百万円/人` |
| 対象期 | 数値 | ヘッダの開始〜終了期の範囲内 | `55` |
| 年度目標 | 数値 | その年度の目標(線形補間値・修正可) | `15` |
| 最終目標 | 数値 | 当該中計 最終年度の到達目標 | `15` |
| 算出方法 | 単一選択 | 線形補間/手動設定(行ごと上書き可) | `線形補間` |
| 備考 | テキスト | KGI定義(ROA=経常利益÷総資産 等) | |

> ROA = 経常利益 ÷ 総資産、労働生産性 = 控除法付加価値 ÷ 人員。算出に必要な勘定は `KAIKEI_ACTUAL` の入力科目に含める(`07_kpi-layering.md` 確定事項)。
> **線形補間の汎用化**: 起点 = 中計開始期の値(`COMPANY_KPI` の当期目標 or 前中計の着地)、終点 = 終了期のKGI。スパンは任意(3カ年に固定しない)。複数中計が連続する場合、前中計の最終年度を次中計の起点に引き継ぐオプションを持つ。

#### 2.0.2 `COMPANY_KPI`(既存流用 / 全社年度PL計画)

既存テーブル(`lib/lark-tables.ts` の `COMPANY_KPI_FIELDS`)を**そのまま流用**。当期の年度目標(売上/原価/販管費/営業利益/限界利益/経常利益/製造原価率/外注発注率/人員計画 等)を保持。**実績は持たず**、`KAIKEI_ACTUAL` と突き合わせて進捗算出。

> 不足するKGI算出用フィールド(総資産・付加価値構成科目)は持たせず、`KAIKEI_ACTUAL` 側で管理する。

#### 2.0.3 `KAIKEI_ACTUAL`(会計データ実績)

**本部長が一括入力**する会計実績。粒度可変(月/四半期/半期)。L0・L1・生産本部Lv2 が共通参照(二重入力排除)。**旧 `SEISAN_KPI_BASIS` を統合**。

| フィールド名 | 型 | 説明 | 例 |
|-------------|----|----|----|
| 実績ID | テキスト | `期-科目-期間` | `50-売上高-202508` |
| 期 | 数値 | | `50` |
| 粒度 | 単一選択 | **月 / 四半期 / 半期** | `月` |
| 期間 | テキスト | 月=`2025-08` / 四半期=`Q1` / 半期=`上期` | `2025-08` |
| 会計月序 | 数値 | 月粒度時の並び(8月=1〜7月=12)。粗粒度は範囲開始月 | `1` |
| 勘定科目 | 単一選択 | 売上高/製造原価/販管費/営業利益/経常利益/総資産/材料金額/人員数/人件費/賃借料/租税公課/純金融費用/減価償却費 | `売上高` |
| 実績値 | 数値 | | `196.6` |
| 単位 | テキスト | 百万円/人 等 | `百万円` |
| 入力者 | テキスト/担当者 | (運用上は本部長) | |
| 入力日時 | 日時 | | |
| 確定フラグ | チェックボックス | 経理確定でロック | `true` |

> **粒度の正規化**: 月別が入っていれば四半期/半期は自動集計。粗粒度しか無い科目はその粒度で年度累計に算入。生産本部Lv2の粗利率(M-01)等は売上高・製造原価をここから取得して算出。
> **勘定科目セット**: 控除法付加価値(経常利益+人件費+賃借料+租税公課+純金融費用+減価償却費)の構成科目を含む。**最終確定は要相談**(`07_` ❓1)。

### 2.4 ~~`SEISAN_KPI_BASIS`(月次基礎データ)~~ → **廃止・`KAIKEI_ACTUAL` に統合**

> **確定(`07_kpi-layering.md`)**: 基礎データ算出KPI(粗利率・総資産回転率・材料金額比率)の入力元は、独立テーブルにせず **経営レイヤーの `KAIKEI_ACTUAL`(§2.0.3)に一本化**。本部長が「経営」エリアで一括入力した会計データを、生産本部Lv2 が参照して算出する(二重入力排除)。

- 粗利率(M-01) = Σ(売上高−製造原価) / Σ売上高
- 総資産回転率(M-02) = 売上高(年換算 = 累計/経過月×12) / 直近総資産
- 材料金額比率(M-92) = Σ材料金額 / Σ製造原価

> いずれも `KAIKEI_ACTUAL` から該当科目を取得し read 時に算出(算出ロジックは `04_api-design.md` §2.5)。

### 2.5 `SEISAN_KPI_MEASURE`(施策ヘッダ)

Excel `03_施策管理` の「重点施策」。部署グループ × 期で複数。

### 2.6 `SEISAN_KPI_PDCA`(施策 月次PDCAログ)

**1レコード = 1施策 × 1対象月**。

| フィールド名 | 型 | 説明 |
|-------------|----|----|
| PDCA_ID | テキスト | `施策ID-202508` |
| 施策ID | テキスト | → MEASURE 連結 |
| 期 | 数値 | |
| 対象年月 | テキスト | |
| 会計月序 | 数値 | 1〜12 |
| 計画_Plan | テキスト(長文) | 責任者記入 |
| 実施_Do | テキスト(長文) | 責任者記入 |
| 対象KPI実績(当月) | 数値 | 対象KPIの当月実績(`SEISAN_KPI_ACTUAL` から自動取込) |
| 効果_自動判定 | 単一選択(自動) | システムが算出する効果の目安(下記基準。改善/横ばい/悪化) |
| 効果_Check | 単一選択 | 責任者が確定する効果(自動判定を初期値に手動上書き可) |
| 効果_メモ | テキスト(長文) | 自由記述(自動判定と異なる場合の根拠等) |
| 本部長コメント | テキスト(長文) | 会議で本部長記入 |
| 翌月アクション | 単一選択 | 継続/強化/見直し/完了 |
| 記入者 | テキスト | |
| 更新日時 | 日時 | |

> 色分け(計画=青/実施=水/効果=黄=会議前 / コメント=橙/アクション=緑=会議中)は UI 側の表現。データには持たない。

#### ④ 効果あり/なしの判定基準(明確化)

効果は「**対象KPIの実績が、施策の良い方向に動いたか**」で判定する。`基準値`(施策開始時)・前月実績・KPIの`良い方向`を使い、**システムが自動で目安を提示 → 責任者が確定**(手動上書き可)。

| 効果 | 自動判定の基準(対象KPIの`良い方向`を考慮) |
|------|------------------------------------------------|
| 🟢 **改善** | 当月実績が 基準値(or 前月)より **良い方向に 5%以上** 変化、**または** 判定ランクが上昇(赤→黄→緑) |
| ⚪ **横ばい** | 変化が **±5%以内**、かつ 判定ランク変わらず |
| 🔴 **悪化** | 良い方向と逆に **5%以上** 変化、**または** 判定ランクが低下 |

> - **比較対象の既定**: 「施策開始時の基準値」を基準にした累積効果。月次の増減を見たい場合は「前月比」に切替可(画面オプション)。
> - **5%閾値はパラメータ化**(KPIの性質で調整可。`SEISAN_KPI_MASTER` 備考 or 設定で持つ)。
> - **会議では対象KPIの実績推移グラフを併記**し、自動判定の根拠を可視化(主観の排除)。
> - 自動判定はあくまで目安。最終確定は責任者(`効果_Check`)。乖離時は `効果_メモ` に理由を残す。

### 2.8 `SEISAN_KPI_HISTORY`(過去実績 43-49期)

Excel `06_過去実績`。目標妥当性検証用の参照値。**部署別・グループ別の抽出に対応**するため、部署/KPI連結を持たせる。

| フィールド名 | 型 | 説明 |
|-------------|----|----|
| 履歴ID | テキスト | |
| 指標名 | テキスト | `鉄工全体生産量` / `クレーム件数` 等 |
| KPI_ID | テキスト(任意) | 対応する現行KPI(あれば。`M-33` 等)→ 部署別抽出のキー |
| 部署ID | テキスト(任意) | Lark `open_department_id`(部署別実績がある場合)→ グループ別集計に使用 |
| 部署 | テキスト(任意) | 部署名(`本社鉄工課` 等) |
| 集計レベル | 単一選択 | 全社/部門/部署(抽出スコープの判別) |
| 単位 | テキスト | |
| 期 | 数値 | 43〜49 |
| 実績値 | 数値 | |

> **過去実績の2つのデータ源**:
> 1. **当システム運用期(50期〜)** — `SEISAN_KPI_ACTUAL`(部署×KPI×月)が蓄積されるため、**部署別は直接、グループ別は `GROUP_MEMBER` 経由で所属部署を合算**して抽出可能。期を重ねるほど部署別の時系列が充実。
> 2. **移行レガシー(43-49期)** — Excel 06 は全社/部門の集約指標が中心。部署別実績が残っている指標は `部署ID`/`KPI_ID` を付与して部署別抽出に対応。残っていない集約指標は `集計レベル=全社/部門` として全社ビューでのみ表示。

> **抽出スコープ**: 過去実績参照は **全社・部門 / 部署別 / グループ別** の3スコープで照会する(`03_screens-and-features.md` ⑥)。グループ別は所属部署(M:N)の値を合算(累計系)/平均(率系)して表示。

## 3. リレーション(ER 概念図)

```
【経営レイヤー L0/L1】
KEIEI_MIDTERM_PLAN (中計3カ年KGI) ──目標──▶ 中計進捗(売上/ROA/労働生産性)
COMPANY_KPI (全社年度PL目標) ──┐
                                ├─ 突合 ──▶ 全社KPI進捗
KAIKEI_ACTUAL (会計実績/本部長一括) ──┘
        │
        └─ 参照(売上高/製造原価/総資産/材料金額) ──▶ 生産本部Lv2 M-01/M-02/M-92 を算出

【生産本部レイヤー L2】
SEISAN_KPI_PERIOD (期)
   │ 1   ── パーティションキー(期)で全テーブルを跨ぐ ──
   ▼ n
SEISAN_KPI_MASTER (KPI定義/部署) ──1───n── SEISAN_KPI_ACTUAL (月次実績/部署単位)
   │ 1                                          │
   │ (部署)                                     └─ 派生(read時): 年累計/平均/直近月値・進捗率・判定・★
   │
SEISAN_KPI_GROUP (グループ) ──1──n── SEISAN_KPI_GROUP_MEMBER (M:N) ──n──1── 部署
   │  ▲ 1部署が複数グループに重複所属(例: 北関東鉄工課 ∈ 鉄工課G & 北関東工場G)
   │ 1 (グループID)
   ▼ n
SEISAN_KPI_MEASURE (施策/グループ単位) ──1───n── SEISAN_KPI_PDCA (月次PDCA)
                                          └─ 効果は対象KPI(ACTUAL)の変化から自動判定

SEISAN_KPI_STAR_ADJ (★手入力) ──加算──▶ ★達成表の「総合計★数」
SEISAN_KPI_HISTORY (43-49期) … 参照のみ
SEISAN_KPI_AUDIT … 全更新を記録
```

> **グループは表示・施策のレンズ**: 実績は部署単位で1回登録、グループ表示時は `GROUP_MEMBER` 経由で所属部署の実績を参照(重複"表示"・複製なし)。

> Lark Bitable は厳密な FK を持たないため、連結は **`KPI_ID` / `期` / `施策ID` の値一致**で運用(既存 menu-permission の `target_id` 連結と同様の方式)。

## ❓要確認(データモデル分)

> ✅ 解決済み: 会計データの一本化(`SEISAN_KPI_BASIS` → `KAIKEI_ACTUAL` 統合)、経営レイヤー2テーブルの追加(`07_kpi-layering.md` で確定)。

1. **派生値のスナップショット保存可否**: 監査要件(確定値の凍結)を考えると、月次確定時に「判定・★・年累計」をスナップショットとして ACTUAL に書き戻す案も有力。read時算出のみで足りるか要判断。
2. **施策グループの粒度**(再掲): MEASURE.`施策グループ` のリスト確定が必要。
3. **Lv2/Lv3 実績の入力方法**: Excel では Lv3「製造部全体」等にも実績行がある。これを Lv4 合算の**自動算出**にするか、**手入力**(Excel 同様)にするか。自動化すると入力負荷減・整合性向上だが、Excel の手入力値と差異が出る可能性。
4. **対象年月の型**: 日付型(月初日)で持つか、`期+会計月序` のみで持つか。トレンド/フィルタの実装容易性で決定。
5. **`KAIKEI_ACTUAL` の勘定科目セット最終確定**: 控除法付加価値の構成科目(`07_` ❓1)に直結。経理の科目定義に合わせる。
6. **粗粒度(四半期/半期)入力時の月次按分**: 生産本部Lv2 の月次判定が必要な科目で、四半期/半期しか無い場合の月次補間ルール(均等按分 or 当該粒度のみ表示)。
