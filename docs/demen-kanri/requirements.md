## 1. 背景・目的

外注施工業者から提出される**請求書の「人工（にんく）」**と、**現場作業日報の「出面（作業人数）」**が
一致しているかを確認したい。現状は突合が手作業のため、現場作業日報のデータを集計・出力する
**出面管理機能**を工務課メニューに追加する。

- **データソースは既存の現場作業日報**（`現場作業日報` テーブル `tbl428UNr7jZbN5D` / `LARK_TABLE_NIPPOU`）。
  新規の入力機能は作らず、蓄積済みの日報を抽出・集計・出力する**閲覧/レポート機能**とする。
- 「請求書の人工」＝一般に「延べ作業人数（作業人数の合計）」に相当。これを日報側の集計値と対比する。

## 2. スコープ

- 対象: 現場作業日報に投稿済みのデータの**抽出・集計・Excel出力**。
- 対象外: 出面（作業人数）自体の入力・修正（＝現場作業日報フォームで実施済み）、請求書データの取込・自動照合。
  （請求書との突合は、本機能の出力値を人が請求書と見比べる運用。自動照合は将来拡張。）

## 3. データソースと項目マッピング

抽出元 = `現場作業日報`（`tbl428UNr7jZbN5D`）。本機能で使用する項目:

| 画面項目 | 現場作業日報フィールド | 備考 |
|---|---|---|
| 外注業者名 | `会社名` | 日報投稿時に業者/報告者が記入した会社名。抽出条件（部分一致）・集計キー。**§8-1で確定要** |
| 製番 | `売約番号` | |
| 製番名 | `物件名` | 売約情報からの物件名（受注件名） |
| 作業日 | `作業報告日` | 日付。抽出条件（From-To）・ソート・日数集計の基準 |
| 作業人数 | `作業人数` | 数値。人数合計（＝人工相当）の集計対象 |
| （参考）報告者 | `報告者氏名` | 明細表示に含めるかは §8-4 |

## 4. 機能要件

### F-01 抽出条件
- **作業日 From / To**: 両方**必須**。`作業報告日` が From〜To（両端含む）の範囲のレコードを対象。
  - From > To の場合はエラー表示。
- **外注業者名**: **部分一致**・**任意**（未入力可）。`会社名` に指定文字列を含むレコードに絞り込む。
  - 未入力時は業者で絞り込まず全業者を対象に出力する。

### F-02 表示単位の切替（明細 / 集計）
- **明細単位** と **施工業者ごとの集計単位** を**区分（ラジオ）またはチェックボックス**で切替可能とする。
  - 明細単位: 日報レコード1件＝1行。
  - 集計単位: 外注業者ごとに1行へ集約。

### F-03 ソート順
- **集計単位**: `外注業者名` 昇順。
- **明細単位**: `外注業者名` → `製番` → `製番名` → `作業日` の昇順（この優先順）。

### F-04 集計項目（合計）
条件一致したデータについて、**作業日数の合計**と**作業人数の合計**を算出・表示する。

- **作業人数合計** = 対象レコードの `作業人数` の総和（延べ人数＝請求書の人工と対比する値）。
- **作業日数合計** = 対象の作業日数。**定義は §8-2 で確定要**（候補: (a) 日報レコード件数、(b) 実働日数＝`作業報告日` のユニーク日数）。
- 集計単位: 各外注業者行に「作業日数」「作業人数」を表示し、最下部に**全体合計**行。
- 明細単位: 各明細行に `作業人数`（および `作業日`）を表示し、最下部に**全体合計**（作業日数合計・作業人数合計）行。

### F-05 Excel表形式の表示・ダウンロード
- 抽出・集計結果を**Excel表形式で画面表示**し、**Excel（.xlsx）ファイルとしてダウンロード**可能とする。
  （実装: 既存導入済みの `xlsx` を使用。ファイル名例: `出面管理_{From}_{To}.xlsx`。）
- 画面表示とダウンロード内容は一致させる（同じ列・並び・合計）。

#### 出力列（案）
- **明細単位**: 外注業者名 / 製番 / 製番名 / 作業日 / 作業人数 （＋報告者？§8-4）／ 末尾に合計行（作業日数合計・作業人数合計）
- **集計単位**: 外注業者名 / 作業日数 / 作業人数 ／ 末尾に合計行

## 5. 画面要件（レイアウト案）

```
[出面管理]  ← 工務課メニュー配下
┌───────────────────────────────────────────────┐
│ 作業日 [From▼] 〜 [To▼](必須)  外注業者名 [____](部分一致)  │
│ 表示単位: (●)集計  ( )明細        [検索]  [Excelダウンロード] │
├───────────────────────────────────────────────┤
│  （結果テーブル：Excel表形式）                              │
│  外注業者名 | 作業日数 | 作業人数                            │
│  ...                                                        │
│  合計       |   xx     |   xx                               │
└───────────────────────────────────────────────┘
```

- 検索前は結果空。From-To未入力での検索は不可（バリデーション）。
- 結果多数時のページング/件数上限は §8-5。

## 6. 非機能・権限・メニュー配置

- **配置**: 工務課メニュー配下に「出面管理」を追加。`MS_SYSメニュー表示マスタ` + `MS_SYS機能配置マスタ` に登録し、
  `program_id` を新規採番。サーバ側でメニュー権限を強制（`lib/menu-access.ts` の方式）。
- **権限対象**: 工務課（＋必要に応じ管理者）。閲覧・出力のみ（データ更新なし）。
- **認証**: 既存の社内ログイン（middleware保護下の `/api/*`）。
- **性能**: Lark Base 抽出はサーバ側（Amplify 28秒制約に留意。期間・件数に応じ集計はサーバで実施）。

## 7. 開発対象（想定・確定後に詳細化）

- 画面: 工務課配下の新規ページ（例 `app/koumu/demen/page.tsx`）。
- API: 抽出・集計エンドポイント（例 `app/api/demen`）。`作業報告日` 範囲 + `会社名` 部分一致で `現場作業日報` を取得し、
  明細/集計・ソート・合計を算出して返す。安全なフィルタ生成（値エスケープ `escapeLarkFilterValue`）。
- Excel: 既存 `xlsx` でサーバ生成 or クライアント生成（施工業者別DL実装と同様の方式を流用可）。
- メニュー/権限マスタ登録。

## 8. 未確定・要確認事項

1. **「外注業者名」の定義**: 日報の `会社名`（投稿者記入）で確定か。案件マスタの `業者`（登録名）と分けるか。
   請求書突合の観点では実際の出面記録である `会社名` が妥当と想定するが要確認。
2. **「作業日数」の定義**: (a) 日報レコード件数 / (b) 実働日数（`作業報告日` のユニーク日数）。
   同一業者・同一日に複数現場（製番）や複数投稿がある場合の扱い。
3. **同一業者名の表記ゆれ**: `会社名` が自由記入だと表記ゆれで集計が割れる恐れ。名寄せ要否。
4. **明細の表示列**: 報告者氏名・特記事項などを含めるか（請求突合に必要な最小列に絞るか）。
5. **件数上限・ページング**: 期間が長い場合の取得上限（Lark pageSize）・表示上限・パフォーマンス。
6. **無効データの扱い**: 受付コード照合結果/有効フラグは廃止済み。全投稿を対象にするか、除外条件が要るか。
7. **製番名**: `物件名`（=受注件名）でよいか。
8. **出力粒度の併記**: 集計行から明細へのドリルダウン要否（初版は不要想定）。
