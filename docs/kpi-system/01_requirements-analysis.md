# 01. 要件分析サマリー

要件定義書(docx)と KPIマスタ(xlsx 全8シート)の解析結果。設計の前提となる「事実」を整理する。

---

## 1. Why — このシステムで実現したいこと

| # | 狙い | 課題(現状) | システムでの解決 |
|---|------|------------|------------------|
| 1 | 経営目標と現場KPIをつなぐ | 中計(売上67億・ROA13%・労働生産性10百万円/人)と現場数字の関係が見えない | 4階層に分解し「どの数字が経営にどう効くか」を可視化 |
| 2 | 同じ数字を全員が見る | Excel は同時編集・履歴管理が困難、最新版が曖昧 | 1システムに集約・全員が同一最新値・更新履歴記録 |

## 2. What — 何を管理するか

### 2.1 4階層 × 計63指標

| 階層 | 管理対象 | 主な責任者 | KPI例 |
|------|---------|-----------|-------|
| 経営 | 全社 | 社長・役員 | 売上・ROA・労働生産性 |
| **Lv2** | 生産本部 | 本部長 | 粗利率・総資産回転率・全社品質 |
| **Lv3** | 製造部 / 生産管理部 | 部長・次長 | リードタイム・生産効率・在庫 |
| **Lv4** | 6工場課 + 3間接課 | 課長 | 各課のクレーム・不具合・生産量 |

**組織構成(KPIマスタ 07シート より確定)**

- 製造部(6課): 本社鉄工課 / 第二鉄工課 / 北関東鉄工課 / 本社縫製課 / 北多久縫製課 / 北関東縫製課
- 生産管理部(3課): 調達課 / 生産管理課 / 検査課

### 2.2 KPIで管理する情報

- 年間目標 と 月割目標(月次目標換算)
- 月次実績値(**8月〜翌7月** の会計年度)
- 達成判定(**緑 / 黄 / 赤** の3段階)
- 月間目標達成で得る「**★**」と各課の年間獲得★数

### 2.3 施策(KPI改善のPDCA)

- 対象: **Lv3部署**(生産管理部・製造部 鉄工課・縫製課・北関東工場)単位で月次の重点施策
- 記録項目: 施策名 / 対象KPI / 月次の 計画(Plan)・実施(Do)・効果(Check) / 本部長コメント / 翌月アクション
- 効果の選択肢: **改善 / 横ばい / 悪化**
- 翌月アクションの選択肢: **継続 / 強化 / 見直し / 完了**

### 2.4 KPI分類(カテゴリ・集計タイプ・方向)

Excel 07_KPIマスタ から抽出した「型」。判定/★算出ロジックの基礎になる。

- **カテゴリ**: 品質 / 納期 / 原価 / 安全
- **集計タイプ**:
  - `累計` … 月次を積み上げ(例: クレーム件数、生産量)
  - `平均` … 経過月平均(例: リードタイム、生産効率)
  - `直近月値` … 最新月の値が現在値(例: 在庫+仕掛金額)
  - `基礎データ算出` … 率を直接入力せず基礎データから算出(粗利率・総資産回転率・材料金額比率)
- **良い方向**: `高い方が良い` / `少ない方が良い`

### 2.5 基礎データ算出KPI(3指標)

率を直接入力せず、基礎データを月次入力して**累計ベースで自動算出**する。

| KPI | 算出式 | 必要な基礎データ |
|-----|--------|------------------|
| 粗利率(M-01) | (売上高 − 製造原価) ÷ 売上高 | 売上高・製造原価 |
| 総資産回転率(M-02) | 売上高(年換算) ÷ 総資産 | 売上高・総資産 |
| 材料金額比率(M-92) | 材料金額 ÷ 製造原価 | 製造原価・材料金額 |

## 3. Who — 利用者(6ロール)

| ロール | 該当者 | 主な権限 |
|--------|--------|----------|
| 経営層 | 社長・役員 | ダッシュボード閲覧中心 |
| 本部長 | 生産本部長 | 全KPI確認 / 施策に本部長コメント / 翌月アクション決定 / 基礎データ入力 |
| 部長・次長 | 製造部・生産管理部の部長/次長 | 自部門の施策(計画・実施・効果)入力 |

| 閲覧者 | 関連部門・スタッフ | ダッシュボード/達成状況の閲覧のみ |
| 管理者 | DX推進室 | KPI定義編集 / 期切替 / ユーザー管理 |

## 4. When / Where — 運用サイクル(月次)

| 手順 | 誰が | 何をする | 入力タイミング |
|------|------|---------|----------------|
| 1 | 担当(生産管理課) | 前月実績値を入力 | 翌月(前月実績は当月10日前後確定) |
| 2 | 各部責任者 | 施策の計画・実施・効果(Check)記入 | 月次 |
| 3 | 本部長 | ダッシュボード確認・本部長コメント記入 | 月次会議前 |
| 4 | 本部長+各責任者 | 会議で翌月アクション決定・記録 | 月次会議 |
| 5 | 本部長 | 経営KPI(粗利率等)確定値入力 | 翌々月(経理確定後) |

- **通常KPI**(品質・納期・生産性・安全): 「翌月」入力
- **経営KPI**(粗利率・総資産回転率・材料金額比率): 「翌々月」入力(経理確定待ち)
- 会議では**画面投影**して使用。

## 5. How — 必要機能(全7機能)

| # | 機能 | 概要 |
|---|------|------|
| ① | ダッシュボード | 経営KPI・全KPI健全性・各部★達成・施策進捗・月次トレンドを1画面集約。会議投影用 |
| ② | KPI実績入力 | 月次実績入力。入力即判定(緑黄赤)表示 |
| ③ | 施策管理(PDCA) | 部署ごと重点施策と月次PDCA。対象KPI選択で実績/目標/判定を自動表示 |
| ④ | ★達成評価 | 月間目標達成で★1 / 期末累計達成で各項目★+3。製造部6課ランキング |
| ⑤ | KPIマスタ管理 | KPI定義・目標値管理(管理者・期切替時) |
| ⑥ | 過去実績参照 | 過去7期分(43-49期)実績参照。目標妥当性検証 |
| ⑦ | データエクスポート | KPI実績・施策ログを CSV/PDF 出力 |

### 5.1 達成判定ルール(緑/黄/赤)

要件定義 §5.2(簡易版)+ KPIマスタ 00_運用ガイド(詳細版)。**正本は Excel**。

**要件定義 §5.2(原則)**
- 🟢 緑: 目標達成(達成率95%以上、または年間ペース内)
- 🟡 黄: あと一歩(達成率80〜95%)
- 🔴 赤: 未達(達成率80%未満)

**00_運用ガイド(集計タイプ別の詳細・正本)**

| 型 | 緑 | 黄 | 赤 |
|----|----|----|----|
| 少ない方が良い(累計/件数系) | 月次目標以下 | 月次目標の100〜150% | 150%超 |
| 直近月値(在庫+仕掛) | 達成率95%以上 | 80〜95% | 80%未満 |
| 外注金額(累計 vs 月割合算) | 目標達成 | 目標の100〜110% | 110%超 |
| 高い方が良い・平均型(粗利率等) | 目標達成 | 目標の95〜100% | 95%未満 |

> ⚠️ 型ごとに閾値・対比基準が異なる。判定ロジックは「集計タイプ × 方向」で分岐する共通関数として実装し、Excel の値と突合して検証する(→ `04_api-design.md`)。

### 5.2 ★達成ルール(04_/05_製造部・間接部門★達成表 より)

- 月間目標達成で **★1個**
- 年間目標を**期末累計**で達成すると **★+3**(期末のみ)
- **5S大賞 / 労災**は手入力で★を増減(例: 5S大賞 +1〜+3 / 労災 −2〜−5)
- 集計対象項目は部署ごとに4項目前後(品質クレーム/品質不具合/納期LT/原価生産効率 など)
- 「総合計★数(5S・労災込)」= 自動算出★ + 手入力調整
- 間接部門は「経過月内の空欄も達成扱い」という特例あり

## 6. 品質要件(非機能)

| 区分 | 要件 |
|------|------|
| 使いやすさ | 1画面で経営状況把握 / 入力即判定 / 画面遷移最小 / 会議投影で見やすい |
| 信頼性 | 入力・編集の履歴(誰がいつ何を)を必ず記録 / 月次確定データは原則ロック(管理者承認なしに変更不可) |
| セキュリティ | ロールごとに閲覧・編集範囲を制御 / 経営KPIは閲覧権限管理 |
| 拡張性 | 組織変更(課の新設・統廃合)に柔軟対応 / 将来 会計システムから基礎データ自動取込できる構造 |

## 7. その他(移行・進め方)

- **データ移行**: KPI定義(63指標)・50期実績・43-49期過去実績を初期データとして移行
- **並行運用**: 移行後の値が現行 Excel と一致することを 1〜2ヶ月の並行運用で確認
- **進め方の希望**:
  - 最初に画面イメージ(モックアップ)で使い勝手をすり合わせ
  - 月次レビュー会議の実際の使われ方を見学した上で設計反映
  - 不明点・懸念は早期共有、生産本部側との対話で詰める

## ❓要確認(本ドキュメント分)

1. **会計年度の基準月**: 8月開始で確定(Excel・要件一致)。期表記は「50期」。
2. **「北関東工場」の粒度**: 施策管理(03シート)は Lv3 単位で「製造部 鉄工課/縫製課/北関東工場」とまとめている一方、★達成表・KPIマスタは6課個別。施策の単位は **5グループ**(生産管理部 / 製造部鉄工課 / 製造部縫製課 / 北関東工場 / ※生産管理課系)で良いか要確認。
3. **課長ロール**: 要件 §3 に「実装できれば」。スコープに含めるか(含める前提で権限設計、機能フラグでON/OFF可能に)。
4. **労働生産性・売上67億・ROA13%** 等の全社経営KPI: 本システムは生産本部スコープ。全社値は既存 `COMPANY_KPI` テーブルと重複しうる → 連携/参照方針を要確認(→ `05_`)。
