# 07. KPI階層の整理 — 中期経営計画 / 全社KPI / 部署別KPI

「何をどのレンズで見るか」を明確に分離する。特に **経営の達成度(会計データ駆動)** と **現場のオペレーション(生産本部の63指標)** を別エリアとして扱う。

---

## 1. 3つのレンズ(視点)の定義

| レンズ | 時間軸 | 見るもの | 数字の源 | 主たる読者 |
|--------|--------|---------|----------|-----------|
| **L0 中期経営計画**(戦略) | **3カ年(50→52期)**※確定 | 中計KGI(売上67億・ROA13%・労働生産性10百万円/人) への到達度 | 会計データ(実績) | 社長・役員・本部長 |
| **L1 全社KPI**(年度計画) | **単年度**(当期) | 当年度のPL計画(売上/原価/販管費/営業利益/限界利益/経常利益…)の進捗 | 会計データ(実績) | 経営層・本部長 |
| **L2 生産本部KPI**(部署別/現場) | **月次** | 生産本部(製造部・生産管理部)の現場KPI 63指標(品質/納期/原価/安全) | 現場の月次実績 | 本部長・部長・課長 |

> ⚠️ **設計上の分離**: L0/L1 は「**会計データを入れると経営計画への進捗が出る**」エリア。L2 は「**現場が月次実績を入れると判定・★が出る**」エリア。両者は別画面・別テーブルとし、**接続点**(下記§4)だけで繋ぐ。

## 2. 各レンズで見るもの(一覧)

### L0 中期経営計画(複数年・KGI)

| 指標 | 定義 | 必要な会計データ |
|------|------|------------------|
| 売上高(中計目標 67億) | 年度売上 | 売上高 |
| ROA(13%) | **営業利益 ÷ 総資産**※確定 | 営業利益・総資産 |
| 労働生産性(10百万円/人) | **付加価値(控除法) ÷ 人員**※確定 | 経常利益+人件費+賃借料+減価償却費等・人員数 |
| (任意)粗利率・自己資本比率 等 | 中計に掲げる他のKGI | 各勘定 |

> 控除法(日銀方式)の付加価値 = 経常利益 + 人件費 + 賃借料 + 租税公課 + 純金融費用 + 減価償却費。算出に必要な勘定は `KAIKEI_ACTUAL` の入力科目に含める。

- **見せ方**: 中計初年度〜最終年度の目標トラジェクトリに対し、各年度の実績(=会計データ集計)をプロット。「あと何%」「年次ペース内か」を表示。

### L1 全社KPI(単年度・PL計画)

> 既存 `COMPANY_KPI` テーブルが**ほぼそのまま該当**(目標値を保持済み)。これに**実績(会計データ)**を突き合わせて進捗を出す。

| 区分 | 指標(既存 COMPANY_KPI_FIELDS) |
|------|-------------------------------|
| 売上 | 売上目標 / 月次売上目標 |
| PL | 売上原価・販管費・営業利益・経常利益(各 目標/率) |
| 限界利益 | 変動費・限界利益・固定費(各 目標/率) |
| 製造・外注 | 製造原価率・実行予算率・外注発注率 |
| その他 | 人員計画・設備投資・広告販促費 |

- **見せ方**: 年度計画 vs 実績累計(会計データ)。月次/四半期/半期で進捗率と着地見込み。

### L2 生産本部KPI(部署別・現場の63指標)

> `02_data-model.md` の `SEISAN_KPI_*`。**この案件の主対象**。

| 階層 | 部署 | 見るもの |
|------|------|---------|
| Lv2 | 生産本部全体 | 粗利率・総資産回転率・材料金額比率・全社クレーム/不具合・在庫+仕掛・外注金額 |
| Lv3 | 製造部 / 生産管理部 | リードタイム・生産効率・在庫・部門品質 |
| Lv4 | 6製造課 + 3間接課 | 各課のクレーム・不具合・生産量・生産効率・安全 |

- **見せ方**: 月次実績入力→緑黄赤判定→★達成・施策PDCA(既設計どおり)。

## 4. レンズ間の接続点(どの数字が上位に効くか)

L2(現場)の一部の経営KPIが、L1/L0 の数字と**意味的に対応**する。値は二重入力せず**参照**でつなぐ。

| L2 生産本部(現場) | 対応する L1/L0(経営) | 接続方法 |
|---------------------|------------------------|----------|
| 粗利率(M-01) | 全社粗利/原価率 | 同じ会計データ源(売上高・製造原価)を共有 |
| 総資産回転率(M-02) | ROA の構成要素 | 売上高・総資産を共有 |
| 材料金額比率(M-92) | 製造原価率/実行予算 | 製造原価・材料金額を共有 |
| 在庫+仕掛(M-93)・外注金額(M-94) | 運転資本・原価 | 生産本部側で月次管理 |

> **原則**: 会計データ(売上高・製造原価・総資産・材料金額・人員 等)の**入力は1箇所**(会計データテーブル)。L0/L1/L2-Lv2 はそこを参照して各々の指標を算出する。**二重入力・二重管理を排除**(要件 1.2「同じ数字を全員が見る」)。

## 5. 会計データ入力(粒度: 月 / 四半期 / 半期)→ 中計進捗

ご要望の中核。**柔軟な粒度**で会計実績を入力し、L0(中計)・L1(年度)への進捗を出す。

### 5.1 入力粒度の考え方

| 粒度 | 用途 | 例 |
|------|------|----|
| **月別** | 詳細管理・生産本部Lv2の月次判定と整合 | 8月,9月,… |
| **四半期** | 経理確定が四半期単位の勘定 | Q1(8-10月)… |
| **半期** | 上期/下期でしか出ない勘定 | 上期(8-1月)/下期(2-7月) |

- 1つの勘定科目に対し**いずれかの粒度で入力**できる(科目ごとに粒度を選択)。
- 異なる粒度を**共通の年度累計**に正規化して進捗計算(月→四半期→半期→年度のロールアップ)。
- 月別が入っていれば四半期/半期は自動集計。粗い粒度しか無い勘定は、その粒度で按分 or そのまま累計。

### 5.2 進捗の出し方

```
年度進捗率 = 実績累計(入力済み期間) ÷ 年度目標 × (12 ÷ 経過月数)   // 年次ペース換算
中計進捗   = Σ(各年度実績) ÷ 中計目標(最終年度)                    // 到達度
```

- 月次レビューでは「**年度計画に対して何%・ペース内か**」、四半期/半期レビューでは「**中計トラジェクトリ上のどこにいるか**」を表示。

## ✅ 確定した方針(2026-06-08)

- **L0 中期経営計画 = 3カ年(50→52期)**。最終年度(52期)に KGI(売上67億・ROA13%・労働生産性10百万円/人)到達を目標とし、各年度の目標トラジェクトリを `KEIEI_MIDTERM_PLAN` に保持。
- **L1 全社KPI = 既存 `COMPANY_KPI` テーブルを流用**(目標値)。実績は新 `KAIKEI_ACTUAL` で保持し、突き合わせて進捗を出す。重複入力なし。
- **会計データ = 本部長が一括入力 × 一本化**。`KAIKEI_ACTUAL` に1箇所入力し、L0/L1 と生産本部Lv2(粗利率・総資産回転率・材料金額比率)が共に参照。`SEISAN_KPI_BASIS` は `KAIKEI_ACTUAL` に統合(独立テーブルにしない)。
- **ROA = 営業利益 ÷ 総資産**。
- **労働生産性 = 控除法の付加価値 ÷ 人員**(付加価値 = 経常利益+人件費+賃借料+租税公課+純金融費用+減価償却費)。
- **画面配置 = 独立「経営」メニューを新設**(生産本部KPIメニューとは別系統)。中計・全社KPI・会計データ入力をここに集約。
- **中計トラジェクトリ = 線形補間で自動生成**。50期の起点値(初年度計画値)と52期のKGI(67億/13%/10百万円)を直線按分して 51期の中間目標を代入(各期 個別修正も可能)。
- **会計入力粒度 = 月別を基本、月次で出ない科目(総資産等)は四半期/半期**で入力 → 共通の年度累計に正規化。

## 7.5 中期経営計画の拡張性(複数中計の追加)

**今回の中計は 50→52期。ただし将来 53→55期、56→58期… を追加運用できる仕様**とする(管理者操作のみ・コード改修不要)。

### 実現方法

- 中計を **ヘッダ(`KEIEI_MIDTERM_PLAN_HEADER`)+ 明細(`KEIEI_MIDTERM_PLAN`)** に正規化(`02_data-model.md` §2.0.1a/b)。
  - ヘッダ = 中計1本(中計名・開始期・終了期・ステータス・KGI指標セット)。
  - 明細 = 指標 × 対象期 の目標値(線形補間で自動生成・個別修正可)。
- **任意スパン対応**: 3カ年に固定せず、開始期〜終了期を自由設定。
- **連続運用**: 前中計の最終年度(52期)の着地を、次中計(53→55期)の起点に引き継ぐオプション。
- **期マスタ**(`SEISAN_KPI_PERIOD`)に 53/54/55期… を追加すれば年度運用が継続。
- **画面**: 「経営」ダッシュボードの**中計セレクタ**で、現行/次期/過去の中計を切替表示。「現行」は常に1本。

### 運用イメージ(管理者)

```
1. 中計ヘッダを新規作成 … 中計名「第◯次中計」/開始55期は誤り→ 例: 開始53期・終了55期/ステータス=次期
2. KGI指標セットを選択 … 売上高・ROA・労働生産性(必要なら指標追加)
3. 各KGIの最終目標(55期値)を入力 → 線形補間で53/54期目標を自動生成 → 必要なら個別修正
4. 期マスタに53/54/55期を登録
5. 52期終了後、ヘッダのステータスを 次期→現行 に切替
```

> これにより「**55期までの計画も後から追加できる**」要件(拡張性 要件 §6.4)を満たす。組織変更(課の新設・統廃合)も `SEISAN_KPI_MASTER` の有効フラグ+期で対応(既設計)。
