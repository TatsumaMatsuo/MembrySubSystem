## 作成前に

- アプリ（MembrySubSystem）のロールが「編集可能」のためテーブル作成が API で拒否されました（`1254302 RolePermNotAllow`）。そのため **Lark UI で手動作成**します
- **テーブル名・フィールド名は下記のとおり一字一句そのまま**にしてください。アプリ側は名前でフィールドを参照するため、表記ゆれ（スペース・全角半角）があると動きません
- **単一選択の選択肢は必ずここに書いた文言どおり全部作成**してください。アプリから未登録の選択肢値を書き込むと `1254302 Permission denied` になり、後から API で追加できません
- 作成後、各テーブルの URL から `table=tblXXXXXXXX` を控えて、末尾の「作成後の作業」へ進んでください

凡例: フィールド型は Lark UI の表示名です。

## 作成後の作業

1. 各テーブルの URL から `table=tblXXXXXXXX` を控える
2. 以下のように連絡してください（または `.env` / Amplify 環境変数に設定）

```
LARK_TABLE_TANAOROSHI_PERIOD=tbl........
LARK_TABLE_TANAOROSHI_WH_STATUS=tbl........
LARK_TABLE_TANAOROSHI_ENTRY=tbl........
LARK_TABLE_TANAOROSHI_DIFF=tbl........
LARK_TABLE_TANAOROSHI_REASON=tbl........
LARK_TABLE_TANAOROSHI_AUDIT=tbl........
```

3. 検証は次のコマンドで行えます（フィールド名・型・選択肢が定義どおりかを自動チェック）

```
npx tsx scripts/verify-tanaoroshi-tables.ts
```
