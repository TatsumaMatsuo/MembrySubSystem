## テーブル: `参考図汎用マスタ`

| フィールド名 | 型 | 必須 | 備考 |
|---|---|---|---|
| `区分` | テキスト（単一行） | ✅ | どの検索項目の候補か（下記「区分一覧」の値と完全一致） |
| `値` | テキスト（単一行） | ✅ | プルダウン/検索ポップアップに出る候補値 |
| `表示順` | 数値 | 任意 | 並び順（未設定時は値の昇順） |
| `有効フラグ` | チェックボックス | 任意 | OFF の行は候補から除外（既定は ON 扱い） |

> 突合は `区分`＋`値`。同一(区分,値)の重複は不可。

## 初期値の投入（おすすめ）

候補値を手入力する必要はありません。**現在の台帳データの実値から自動で初期投入**できます
（部品マスタ投入と同じ仕組み: `scripts/sankou-zu/import-hanyou.ts` を用意します）。

1. 上記テーブルを Lark UI で作成 → テーブルID（`tbl…`）を共有
2. こちらで台帳の各列の distinct 値を `区分`/`値` に変換して投入（dry-run→execute、冪等）
3. 以後は Lark 上で値の追加・除外（有効フラグ）・並び替え（表示順）を業務側で運用

## 作成後にいただきたい情報

1. `参考図汎用マスタ` のテーブルID（`tbl…`）
2. サイズ1/サイズ2/F1/F2 を「マスタ化する／部分一致入力にする」のどちら希望か

→ 受領後に `lib/lark-tables.ts` 登録 → 初期投入 → 検索画面の★候補ソースを汎用マスタへ切替。
