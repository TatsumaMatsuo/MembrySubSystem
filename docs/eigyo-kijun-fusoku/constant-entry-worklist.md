# 標高依存積雪 — 計算パターン方式 ステータス

元Excelを計算パターン方式（式テンプレート＋定数）へ移行した案件の進捗記録。

## 現状（2026-06-26 完了）

- 計算パターン: **K001〜K063（63式）** … 当初77式から再集約。多段階段型が `定数1〜19` を参照
- 元データ列: `定数1〜19`＝列Y(24)〜AQ(42)、`計算パターンID`＝列AR(43)、基準値=U(20)、積雪量=R(17)
- Larkマスタ `基準風速・積雪量マスタ`: フィールド `定数1〜19`・`計算パターンID` 追加済（Lark UI手動作成）
- **標高依存 1,450行すべてにパターンID＋定数を設定 → auto算出 100%・式エラー0**

> 評価エンジン `lib/kijun-fusoku-snow.ts`（eval不使用パーサ、定数1〜19対応）で確定算出。
> パターン式は `lib/kijun-fusoku-patterns.ts`（Excel「計算パターン」シートから機械生成）。

## 残課題（データ修正・2行）

検証ツール `validate-snow.ts` が **標高↑で積雪↓の逆転** を2行検出：

| 地域 | pid | 問題 |
|---|---|---|
| 熊本県 上益城郡 甲佐町 | K052 | 70m超で固定20cm（=積雪量R）を返すが、70m以下は25cm。W原文は「70m超は計算式」。**逆転** |
| 熊本県 上益城郡 益城町 | K052 | 同上 |

- K052 = `IF(標高<=基準値,定数1,積雪量)`。この2行は **70m超が計算式型** のため K052 は不適合。
- 正しくは **K053（`IF(標高<=基準値,定数1,ROUNDDOWN(標高*定数2+定数3,定数4))`）等＋係数**。
- → Excel側でこの2行のパターンID・定数を修正し、再import→validate で解消する。

## 反映手順（定数・パターン修正時）

1. Excel の `定数1〜19`（列Y〜AQ）／`計算パターンID`（列AR）を修正
2. `npx tsx scripts/kijun-fusoku/import-to-lark.ts --file <xlsx>`（dry-run）
3. `… --execute`（Larkへ反映）
4. `npx tsx scripts/kijun-fusoku/validate-snow.ts`（auto率・単調性・妥当域を再検証）
