# 08. Lark Base 構築仕様(#56)

Lark Base に**手動で作成**するためのテーブル仕様。`docs/kpi-system/02_data-model.md` を Lark Bitable のフィールド型に落とし込んだもの。

## 0. 事前情報

- **作成先 Base**: **project base**(既存の売約情報などと同じ。env `LARK_BASE_TOKEN` / フォールバック `<project base app_token(env LARK_BASE_TOKEN で設定)>`)
  - ※ 既存 `COMPANY_KPI` もこの project base にあり、**全社年度KPIは流用**(新規作成不要)
- **作成テーブル数**: 13本(経営3 + 生産本部10)
- **Lark フィールド型の対応**

| 本書の表記 | Lark Bitable フィールド型 |
|-----------|--------------------------|
| テキスト | テキスト |
| 数値 | 数値 |
| 単一選択 | 単一選択(オプションは本書記載) |
| 複数選択 | 複数選択 |
| 日付 | 日付 |
| 日時 | 日付(時刻表示ON) or 作成日時/更新日時 |
| チェックボックス | チェックボックス |
| 自動: 作成日時/更新日時 | 作成日時 / 最終更新日時(自動フィールド) |

> **部署について**: 部署マスタは **Lark Contact 部門**を使用。テーブル上は `部署`(テキスト=名称)と `部署コード`(テキスト=`open_department_id`)の2列で保持し、アプリが Lark Contact API で解決する。Bitable のリンク型は使わない。
> **連結(FK)について**: Lark Bitable は厳密なFKを持たないため、`KPIコード` / `期` / `施策コード` / `グループコード` などの**値一致**で連結する(既存 menu-permission と同方式)。リンク型ではなくテキストで持つ。

各テーブル作成後、**テーブルID(`tblXXXX`)を控えて** §99 の env に登録する。

> **テーブル名・フィールド名は日本語**で作成する(業務ユーザーがLark画面で直接見るため)。英語の論理名はコード/env用の識別子として併記。API連携はテーブルIDで行うため、表示名が日本語でも問題ない。

## 100. 作成順の推奨

1. **マスタ系から**: `SEISAN_KPI_PERIOD` → `SEISAN_KPI_GROUP` → `SEISAN_KPI_GROUP_MEMBER` → `SEISAN_KPI_MASTER`
2. **経営マスタ**: `KEIEI_MIDTERM_PLAN_HEADER` → `KEIEI_MIDTERM_PLAN`
3. **トランザクション**: `KAIKEI_ACTUAL` / `SEISAN_KPI_ACTUAL` / `SEISAN_KPI_MEASURE` → `SEISAN_KPI_PDCA` / `SEISAN_KPI_STAR_ADJ`
4. **参照・ログ**: `SEISAN_KPI_HISTORY` / `SEISAN_KPI_AUDIT`

作成後、テーブルIDを §99 に記入してお知らせください → コード側(`lib/lark-tables.ts` 拡張 + データ移行スクリプト #57)に進みます。
