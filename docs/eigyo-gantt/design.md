## 1. スコープ（MVP）

- 独立した「ガントチャート」画面（営業部/支援ツール）。案件横断で作成・保存・一覧・再編集。
- 2ペイン（左=タスクグリッド／右=タイムライン）。タスクの追加/編集/削除/並べ替え、D&Dでバー移動・期間伸縮、日/週/月切替、今日ライン、題名付き保存。
- 作業工程ひな形マスタ（登録/編集/削除/分類）。ひな形＋基準日から自動生成。既存チャート→ひな形化。
- 保存済み一覧・検索（題名/作成者/作成日/案件）、開いて再編集・上書き・別名保存・削除。
- 売約詳細の社内工程表へ、保存ガントを選んで**コピー取込**。

**後回し（次期）**: 依存線(F-07)、PDF/印刷(U-05)、休日カレンダー、同時編集の排他、Excel工程表移行。

## 2. データモデル（Lark, JSONブロブ方式）

要求定義書§6の4テーブル案は、明細をヘッダのJSON列に格納する**2テーブル方式**に簡素化する（100タスク程度ならテキスト列に収まる／保存が原子的／FK結合不要）。一覧・検索に使う項目のみ列として持つ。project base。

### テーブルA: `ガントチャート`（GANTT_CHART）
| 項目名 | 型 | 用途 |
|---|---|---|
| チャートID | テキスト（**主キー**） | 一意ID（アプリ採番 GC-xxxxxxxx） |
| 題名 | テキスト | 一覧/検索 |
| 売約番号 | テキスト | 案件紐付け（横断作成時は空可） |
| 作成者 | テキスト | 検索 |
| 作成日時 | 日付（日時） | 検索/並び |
| 更新日時 | 日付（日時） | 並び |
| データJSON | テキスト | `{unit, from, to, tasks:[{id,order,name,start,end,assignee,progress,pred,notes}]}` |

### テーブルB: `ガントひな形`（GANTT_TEMPLATE）
| 項目名 | 型 | 用途 |
|---|---|---|
| ひな形ID | テキスト（**主キー**） | 一意ID（アプリ採番 TMPL-xxxxxxxx） |
| ひな形名 | テキスト | 一覧 |
| 分類 | テキスト | 分類管理（テント倉庫/膜構造/リニューアル等。単一選択は選択肢追加がAPI不可のためテキスト） |
| 有効フラグ | チェックボックス | 無効化 |
| 更新者 | テキスト | 監査 |
| 更新日時 | 日付（日時） | 並び |
| データJSON | テキスト | `{notes, steps:[{order,name,days,offset,notes}]}`（days=標準所要日数, offset=開始オフセット日数） |

> テーブルはユーザーがLark UIで作成（アプリはスキーマ作成権限なし）。項目名は上記と完全一致必須。IDはenv→フォールバック、無ければ名称解決。

## 3. API（`app/api/eigyo/gantt/…`）

- `charts` GET(一覧/検索: q/seiban/author) / POST(新規・上書き upsert by チャートID) / DELETE(?id=)
- `charts/[id]` GET(1件取得)
- `templates` GET(一覧, 有効のみ) / POST(upsert) / DELETE(?id=)
- `templates/[id]` GET
- 連携は売約詳細側の取込アクション（コピー）で `POST /api/schedule` 相当を再利用（Stage 4で確定）。

## 4. 画面

- `/eigyo/gantt`（一覧＋「新規作成」）→ `/eigyo/gantt/edit?id=`（エディタ）。ひな形管理は `/eigyo/gantt/templates`。
- エディタ: 上部ツールバー（日/週/月・期間From-To・今日へ・ひな形適用・保存/別名保存）、左タスクグリッド、右Frappe Ganttチャート。
- 権限: 一般利用＝ガント作成/保存、マスタ管理＝ひな形編集（`lib/menu-access` + 権限マスタ）。

## 5. ライブラリ

- **frappe-gantt**（MIT）をnpm導入し、`"use client"` で動的import。CSS同梱。Reactラッパで vanilla JS インスタンスを ref 管理。タスク配列⇄Frappe Ganttの相互変換層を用意。

## 6. メニュー登録

- `scripts/register-eigyo-gantt-menu.ts`（sankou-zu方式を踏襲）。営業部 M003 / 支援ツール M003-05 配下に新PGM（自動採番）で `/eigyo/gantt` を追加。表示は権限マスタで別途許可。

## 7. 段階実装計画

1. **基盤**: lark-tables定義、CRUD API（charts/templates）、一覧ページ雛形、メニュー登録スクリプト。テーブル未作成でも安全動作。
2. **エディタ**: frappe-gantt導入、2ペイン・タスクCRUD・D&D・日/週/月・今日ライン・題名保存/別名保存。
3. **ひな形**: ひな形マスタCRUD画面、基準日展開で自動生成、既存チャート→ひな形化。
4. **保存管理の仕上げ＋売約詳細連携**: 検索/削除の仕上げ、売約詳細から保存ガントを選び社内工程表へコピー取込。
5. **（次期）** 依存線、PDF/印刷、休日カレンダー。

各段階で `npx tsc`＋`npm run build` 確認、main と feat/sales-analysis の両ブランチへ反映・検証デプロイ。
